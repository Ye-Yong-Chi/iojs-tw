npm 週報 #21 : npm 3 更新 + react.js 元件工具

訂閱 npm 週報

npm 3 beta 測試者們

如果你是位 npm@3 beta 測試者，在此通知：下載 npm@3 最新及預備版（next versions）的標籤已經更改。現在請改用 npm@3.x-next 及 npm@3.x-latest。詳細請見 CHANGELOG。同樣地，感謝各位給予我們的協助消滅各種臭蟲！

當 Forrest 離開，Kat 加入許多東西到 npm@2，詳細請見 CHANGELOG。

npm 3 beta 給 windows 用戶帶來好消息

Rebecca 告訴 InfoQ 有關 npm@3，特別是安裝程序的改變將會幫助 Windows 用戶。

私有模組的後續檢視

我們已經透過 email 警示私有模組的用戶們，萬一你沒有收到，再次通知，上週有個私有模組的安全性議題。私有模組的 Metadata 洩漏，但不包括整包內容以及私人用戶訊息。詳細請見後續檢視一文。

react podcast 廣播有關 webpack 對上 browserify

如果你正在使用 React.js 元件，你可能會想聽 React Podcast 廣播 Webpack 對上 Browserify，用來管理前端模組相依的工具。長度大約 7 分鐘。

相同主題，Lin 在上個月 jQuerySF 上說明了如何在 npm 使用 Browserify（及 Babel）來管理前端模組，並且將會在八月的 React Rally 演講有關 Webpack。

npm 現在是我的片段資料庫

Sindre Sorhus 在他的 AMA（問我任何事）說明了他如何使用 npm 作為一種用來捕獲（capture）他的程式碼片段的方式。

說明了當你可以 require 它的時後，為何要複製貼上，以及有個清楚意圖的益處。當你在片段中修正一個臭蟲，你只需要更新一個模組，而不是手動修正所有用到片段的地方。
